WAYFINDER LLM CONTEXT

This file provides architectural guidance for AI coding assistants.

------------------------------------------------
ARCHITECTURE OVERVIEW

WCS — WayFinder Client Shell
Win32 application shell responsible for:

- window creation
- input capture
- displaying MUD output
- hosting UI panels (including the map panel)

WMP — WayFinder Map Panel
UI component hosted inside WCS.

Responsibilities:
- render the map produced by WMR
- display the player location
- display discovered rooms

Important:
WMP contains NO navigation or topology logic.

------------------------------------------------
RUNTIME PIPELINE

The runtime flow is:

WCS → WEG → WNE → WMR

WMP is not part of the runtime pipeline.

------------------------------------------------
COMPONENT RESPONSIBILITIES

WEG — WayFinder Event Gateway
- parses MUD text
- emits normalized events

WNE — WayFinder Navigation Engine
- maintains navigation session
- tracks discovered topology
- exposes movement operations

WMR — WayFinder Mapping Runtime
- owns the map model
- performs incremental room placement
- maintains spatial constraints
- triggers solver rebuilds when necessary

------------------------------------------------
DESIGN PRINCIPLES

1. UI must remain thin.
2. Navigation logic must live in WNE.
3. Map logic must live in WMR.
4. Rendering must stay inside WCS/WMP.
5. Solver remains independent.
