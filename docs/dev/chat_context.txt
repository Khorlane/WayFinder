PROJECT: WayFinder
Language: Go
Repository: WayFinder

Purpose:
WayFinder is a MUD navigation client and automapper. It connects to a MUD via
telnet, discovers rooms incrementally, and builds a spatial grid map based on
directional exits.

Rooms are discovered dynamically as the player moves. The map is not preloaded
when connected to a live MUD.

------------------------------------------------
ARCHITECTURE SUMMARY

Subsystems:

WCS — WayFinder Client Shell
WMP — WayFinder Map Panel
WEG — WayFinder Event Gateway
WNE — WayFinder Navigation Engine
WMR — WayFinder Mapping Runtime
solver — constraint/rebuild support used by WMR

Runtime pipeline:

WCS → WEG → WNE → WMR

Notes:
- WMP is a UI panel hosted inside WCS
- WMP is NOT part of the runtime pipeline
- WMR computes layout but does NOT render UI
- WCS currently uses a Win32 implementation
- Full subsystem definitions live in docs/architecture.md
