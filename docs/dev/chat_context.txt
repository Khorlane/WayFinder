------------------------------------------------
PROJECT: WayFinder
Language: Go
Repository: WayFinder (local project, public GitHub repo)

Purpose:
WayFinder is a MUD navigation client and automapper. It connects to a MUD via telnet,
discovers rooms incrementally, and builds a spatial grid map based on directional exits.

The map is NOT preloaded from the world files when used with a real MUD.
Rooms are discovered dynamically as the player moves.

│      - navigation session

│      - topology events
│
├── WayFinder Mapping Runtime (WMR)
│      - incremental room placement
│      - spatial constraints
│      - rebuild logic
│      - layout projection
│
└── WayFinder Map Panel (WMP)
       - hosted inside WCS
       - displays map data produced by WMR

Runtime flow:
WCS → WEG → WNE → WMR

Important:
WMP is a UI panel owned by WCS, not a separate pipeline stage.

------------------------------------------------
KEY DESIGN RULES

Discovery Model:
- rooms are discovered as entered
- exits revealed immediately when room discovered
- no hidden exits exist in this world

Spatial Constraints:
- N/S relationships require same column
- E/W relationships require same row
- ordering must be preserved
- gaps allowed between rooms
- no room allowed between locked adjacency pairs

Example:
D3B above ECE

Valid:

D3B
ECE

D3B
.
ECE

But not:

D3B
NEWROOM
ECE

------------------------------------------------
MAP LAYOUT MODEL

Coordinates are an infinite integer grid.

Rooms contain:

RoomID
Placed flag
Row
Column

Occupancy map tracks which room occupies each cell.

Locks represent discovered adjacencies that must remain valid.

------------------------------------------------
REBUILD SYSTEM

When incremental placement cannot resolve conflicts,
the solver performs a rebuild of the discovered map.

Rebuild process:
1. build constraint set
2. run solver
3. produce new room layout
4. mapper applies results
