# WayFinder Refactor – Step 3 Instructions for Codex

## Objective
Normalize the **input ingestion path** so that all inbound MUD text enters the system through **WTL → WEG**.

Currently the simulator exists in WTL, but the system may still allow direct coupling between simulator code and navigation/mapping logic. This step ensures that the **event interpretation boundary (WEG)** is respected.

Target pipeline:

WTL → WEG → WNE → WMR → WCS

WTL produces **raw MUD text only**.
WEG is responsible for **interpreting that text into navigation events**.

No behavior changes should be visible to the user.

---

## Constraints

- Do NOT create new architectural components.
- Do NOT introduce additional packages.
- Keep repository structure flat.
- Do NOT change the navigation model.
- Do NOT redesign WNE or WMR.
- Do NOT implement live telnet yet.
- Maintain existing simulator behavior.

This step is about **enforcing boundaries**, not rewriting functionality.

---

## Tasks

### 1. Identify any simulator → navigation shortcuts

Review `wtl/simulated_mode.go` and verify whether the simulator:

- directly calls WNE navigation operations
- directly interacts with WMR
- bypasses the event gateway

If any of these exist, refactor so that:

WTL emits **text output** only.

---

### 2. Route simulated output through WEG

Ensure that simulated MUD text generated by WTL is **delivered to WEG** exactly the same way real telnet output will be delivered later.

Conceptually:

WTL(simulated) → raw text → WEG parser → WNE events

Do not introduce unnecessary abstractions. Keep the flow simple.

---

### 3. Preserve existing parsing expectations

If the simulator currently produces text in a format expected by downstream logic, preserve that format.

Do not change simulated MUD output formatting in this step.

---

### 4. Maintain WNE authority

After this step:

- WNE must still own navigation state.
- WNE must still determine room transitions.
- WTL must NOT become authoritative for navigation.

WTL only produces **text**.

---

### 5. Minimal documentation update if required

Only update docs if they incorrectly describe the ingestion path.

Likely files:
- docs/dev/chat_context.txt
- docs/dev/llm_context.txt

Avoid large doc edits.

---

### 6. Regenerate source index

After completion:

Regenerate:

docs/dev/go_source_index.md

This will be used to verify the refactor.

---

## Completion Criteria

Step 3 is complete when:

- WTL produces only raw simulated MUD text.
- WEG is the first interpreter of that text.
- WNE receives navigation events from WEG rather than directly from the simulator.
- WMR remains untouched.
- Project structure remains flat.
- Simulator behavior still works.
- go_source_index.md is regenerated.
